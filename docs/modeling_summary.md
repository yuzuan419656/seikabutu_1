# Modeling Summary

## 1. 目的

レビュー低評価（review_score <= 2）を目的変数とし、
どのような注文が低評価につながりやすいかを予測モデルによって確認した。

---

## 2. 使用データ

分析用テーブル `mart_model_features` を利用した。

目的変数:

- is_low_score
  - 1 : review_score <= 2
  - 0 : review_score >= 3

データ件数:

- 94,453件

低評価率:

- 12.8%

高評価率:

- 87.2%

低評価レビューは少数派であり、
クラス不均衡が存在することを確認した。

---

## 3. 使用した特徴量

使用した主な特徴量は以下の通りである。

### 数値特徴量

- item_count
- total_price
- total_freight
- total_payment
- delivery_days
- delay_days
- freight_ratio
- purchase_weekday
- purchase_hour

### カテゴリ特徴量

- customer_state
- payment_type
- main_category

カテゴリ変数は One-Hot Encoding を実施した。

---

## 4. 学習方法

学習データとテストデータを

- train : 80%
- test : 20%

に分割した。

クラス不均衡へ対応するため、

```python
class_weight="balanced"
```

を利用した。

## 8. EDAとの比較

EDAでは

- 配送遅延が大きい注文ほど低評価率が高い
- 配送日数が長い注文ほど低評価率が高い
- 送料が高い注文ほど低評価率が高い

という傾向が確認された。

モデルの特徴量重要度でも

- delay_days
- delivery_days
- total_freight

が上位に現れており、
EDAで得られた知見と整合的な結果となった。

## 9. 考察

低評価レビューに最も影響していたのは
配送遅延日数および配送日数であった。

配送品質はレビュー評価に大きな影響を与えている可能性が高い。

ただし、
配送品質は配送業者や配送地域の影響も受けるため、
事業者が直接コントロールできる要因ではない。

そのため、

- 配送パートナーの見直し
- 配送遅延が多い地域の監視
- 配送予定日の精度改善
- 配送遅延時の顧客フォロー強化

などが現実的な改善施策として考えられる。

---

## 10. 分析の限界

今回のモデルでは

- 配送日数
- 配送遅延日数

を特徴量として利用した。

これらは配送完了後に判明する情報であるため、
購入時点でのレビュー予測には利用できない。

したがって本モデルは

「低評価の要因分析」

としては有効であるが、

「購入直後に低評価を予測するモデル」

として利用する場合には特徴量の再設計が必要である。
