# Data Quality Policy

## 1. 目的

本ドキュメントでは、レビュー低評価要因分析に使用する分析用テーブル `mart_order_review` のデータ品質確認結果と前処理方針を整理する。

---

## 2. データ概要

分析対象テーブル：`mart_order_review`

| 項目 | 値 |
|------|------:|
| レコード数 | 99,441 |
| カラム数 | 15 |

注文単位（1注文=1行）の分析用テーブルとして構築されていることを確認した。

---

## 3. 注文ステータスの確認

### 確認結果

| order_status | 件数 |
|--------------|------:|
| delivered | 96,478 |
| shipped | 1,107 |
| canceled | 625 |
| unavailable | 609 |
| invoiced | 314 |
| processing | 301 |
| created | 5 |
| approved | 2 |

### 考察

全体の約97%が配送完了（delivered）注文であった。

配送未完了注文では配送日時が存在せず、配送日数や配送遅延日数を算出できない。

### 方針

配送品質とレビュー評価の関係を分析するため、EDAおよびモデリングでは配送完了（delivered）注文を主な分析対象とする。

---

## 4. 欠損値の確認

### 主な欠損列

| 列名 | 欠損数 |
|------|------:|
| order_delivered_customer_date | 2,965 |
| delivery_days | 2,965 |
| delay_days | 2,965 |
| main_category | 2,213 |
| item_count | 775 |
| total_price | 775 |
| total_freight | 775 |
| review_score | 768 |

### 考察

`delivery_days` および `delay_days` の欠損は配送未完了注文に起因するものであり、データエラーではない。

`item_count`、`total_price`、`total_freight` の欠損も主に `canceled` および `unavailable` 注文で発生していることを確認した。

`main_category` の欠損率は約2.2%であり、分析全体への影響は限定的と考えられる。

### 方針

配送未完了注文を除外した後の欠損状況を再確認する。

目的変数である `review_score` が欠損している注文は分析対象から除外する。

---

## 5. 配送日数・配送遅延日数の確認

### 配送日数（delivery_days）

#### 確認結果

| 指標 | 値 |
|------|------:|
| 平均 | 12.1日 |
| 中央値 | 10日 |
| 75パーセンタイル | 15日 |
| 最大値 | 208日 |

#### 考察

配送日数は右裾の長い分布を示しており、一部に非常に長い配送期間を持つ注文が存在した。

レビュー評価別に確認したところ、低評価注文ほど配送日数が長い傾向が確認された。

| review_score | 中央値配送日数 |
|------|------:|
| 1 | 16日 |
| 2 | 13日 |
| 3 | 12日 |
| 4 | 10日 |
| 5 | 9日 |

#### 方針

配送日数はレビュー評価に関連する重要な説明変数候補であるため除外せず保持する。

---

### 配送遅延日数（delay_days）

#### 確認結果

| 指標 | 値 |
|------|------:|
| 平均 | 0.70日 |
| 中央値 | 0日 |
| 75パーセンタイル | 0日 |
| 最大値 | 188日 |

全体の75%以上の注文で配送遅延は発生していなかった。

レビュー評価別の平均遅延日数は以下の通りである。

| review_score | 平均遅延日数 |
|------|------:|
| 1 | 4.53日 |
| 2 | 1.93日 |
| 3 | 0.79日 |
| 4 | 0.29日 |
| 5 | 0.13日 |

#### 考察

配送遅延が大きい注文ほどレビュー評価が低い傾向が確認された。

一方で、レビュー評価1の注文においても中央値は0日であり、低評価の要因は配送遅延だけではない可能性が示唆された。

| review_score | 遅延日数中央値 |
|------|------:|
| 1 | 0日 |
| 2 | 0日 |
| 3 | 0日 |
| 4 | 0日 |
| 5 | 0日 |

#### 方針

配送遅延日数はレビュー評価を説明する重要な特徴量候補として保持する。

---

## 6. レビュー評価の分布

### 確認結果

| score | 割合 |
|------|------:|
| 1 | 11.5% |
| 2 | 3.2% |
| 3 | 8.2% |
| 4 | 19.3% |
| 5 | 57.8% |

### 考察

高評価（4〜5）が全体の約77%を占める。

一方で、低評価（1〜2）は約15%であり、クラス不均衡が存在する。

### 方針

低評価レビューを

`review_score <= 2`

と定義する。

モデリング時は Accuracy のみで評価せず、Precision、Recall、F1-score を併用する。

---

## 7. 今後の分析方針

EDAでは以下の観点を中心に低評価レビューとの関係を確認する。

- 配送日数
- 配送遅延日数
- 商品価格
- 送料
- 商品カテゴリ
- 支払い方法
- 顧客地域

また、要因分析と購入時点予測を区別して考える。

配送日数および配送遅延日数はレビュー要因分析には利用するが、購入時点では利用できないため、購入時点予測モデルでは使用しない。
